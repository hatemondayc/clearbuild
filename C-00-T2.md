## 1. 최초 생성 프롬프트

```
내 아이템을 소개하는 1페이지 웹사이트 index.html을 만들어줘.

- 사업/프로젝트 이름: CLEARBUILD (클리어빌드)
- 한 줄 소개: 재개발·재건축 조합원이 "내가 결국 얼마를 내야 하는가"를 계산해, 동의·매수·보유 판단의 근거를 주는 정비사업 의사결정 도구
- 대표 상품/서비스:
  1. 분담금 시뮬레이터 — 내 물건 정보와 사업 변수로 예상 분담금을 즉시 계산
  2. 총투입비 분석 — 분담금 + 대출이자 + 이주비까지 합친 실제 부담 산출
  3. 시나리오 비교 — 공사비·분양가·용적률이 바뀌면 내 분담금이 어떻게 달라지는지
  4. AI 문서파싱 — 정비계획·관리처분·총회자료(PDF·HWP)를 AI가 읽어 계산 로직 자동 추출
- 연락/신청 방법: 사전신청 이메일 [본인 실제 이메일]

페이지 구성:
1. 첫 화면에 이름과 한 줄 소개를 크게 보여줘. 헤드라인은 "투표 도구는 있다. 판단 근거가 없다."로 크게, 그 아래 한 줄 소개를 서브카피로 넣어줘
2. 대표 상품/서비스 4개를 카드 형태로 보여줘
3. 연락/신청 버튼("분담금 시뮬레이션 사전신청")을 눈에 띄게 넣어줘. 클릭하면 위 이메일로 연결되게 해줘
4. 모바일에서 보기 좋게 만들어줘

디자인:
- 신뢰감 있는 데이터/핀테크 느낌, 과하지 않게 담백하게
- 배경은 밝게, 포인트 컬러는 딥블루 계열
- 버튼은 잘 보이는 색상으로

한글이 깨지지 않게 <meta charset="UTF-8">가 있는지 확인해줘.
```

## 2. 수정 프롬프트 (1회차)

```
방금 만든 index.html을 수정해줘.

수정할 점:
1. 헤드라인 아래 서브카피를 "정비사업 문서를 AI가 읽어, 조합원 개인의 분담금 의사결정 근거를 만듭니다"로 넣어줘
2. 사전신청 버튼 색상과 위치를 더 눈에 띄게 조정해줘
3. 모바일에서 서비스 카드가 세로로 자연스럽게 쌓이게 해줘
4. 전체 톤을 신뢰감 있고 담백하게
```
